CentOS 7.9系统下Docker的在线与离线安装教程

#Centos7.9_docker安装-

卸载旧版本（未安装请忽略）

dnf

​ DNF 是新一代的rpm软件包管理器。他首先出现在 Fedora 18 这个发行版中。而最近，它取代了yum，正式成为 Fedora 22 的包管理器。

​ DNF包管理器克服了YUM包管理器的一些瓶颈，提升了包括用户体验，内存占用，依赖分析，运行速度等多方面的内容。DNF使用 RPM, libsolv 和 hawkey 库进行包管理操作。尽管它没有预装在 CentOS 和 RHEL 7 中，但你可以在使用 YUM 的同时使用 DNF 。

​ DNF 并未默认安装在 RHEL 或 CentOS 7系统中，所以先安装一下dnf。

安装 DNF 包管理器

1、为了安装 DNF ，您必须先安装并启用 epel-release 依赖。

在系统中执行以下命令：

1

yum -y install epel-release

2、使用 epel-release 依赖中的 YUM 命令来安装 DNF 包。在系统中执行以下命令：

1

 yum install dnf

docker-engine安装

###使用 rpm 存储库安装-在线方式

在新主机上首次安装 Docker Engine 之前，需要设置 Docker 存储库。之后，可以安装和更新 存储库中的 Docker。

####设置存储库

安装软件包（提供用于管理 您的 DNF 存储库）并设置存储库。dnf-plugins-core

1
2

sudo dnf -y install dnf-plugins-core
sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

###安装 Docker Engine

1.要安装最新版本：

1

sudo dnf install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

如果系统提示接受 GPG 密钥，请验证指纹是否匹配，如果匹配，请接受它。060A 61C5 1B55 8A7F 742B 77AA C52F EB6B 621E 9F35

此命令会安装 Docker，但不会启动 Docker。它还会创建一个组，但是，默认情况下它不会向该组添加任何用户。docker

2.启动 Docker Engine。

1

sudo systemctl enable --now docker

这会将 Docker systemd 服务配置为在以下情况下自动启动 引导您的系统。如果您不希望 Docker 自动启动，请改用。sudo systemctl start docker

3.通过运行映像来验证安装是否成功：hello-world

1

sudo docker run hello-world

从软件包安装-离线方式

####软件包下载

转到https://download.docker.com/linux/centos/并选择CentOS 版本。然后浏览并下载要安装的 Docker 版本的文件。x86_64/stable/Packages/``.rpm

安装 Docker Engine，将以下路径更改为您下载的路径 Docker 包。

1

sudo dnf install /path/to/package.rpm

Docker 已安装但未启动。组已创建，但没有 用户将添加到组中。docker

启动 Docker Engine。

1

sudo systemctl enable --now docker

这会将 Docker systemd 服务配置为在以下情况下自动启动 引导的系统。如果不希望 Docker 自动启动，请改用。sudo systemctl start docker

从二进制文件安装

下载 Docker 二进制程序包

download.docker.com

这里下载的是 docker-27.5.1.tgz

安装 Docker

将下载好的 Docker 二进制程序包上传到目标机器上，并解压：

1

tar -zxvf docker-27.5.1.tgz

将 Docker 二进制文件移动到 /usr/bin 目录：

1

cp * /usr/bin

After=network-online.target firewalld.service

[Service]

ExecReload=/bin/kill -s HUP $MAINPID
LimitNOFILE=infinity
LimitNPROC=infinity
TimeoutStartSec=0
Delegate=yes
KillMode=process
Restart=on-failure
StartLimitBurst=3
StartLimitInterval=60s

[Install]
